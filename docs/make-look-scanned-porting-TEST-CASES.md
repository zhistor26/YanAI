# PDF 伪扫描懒猫微服移植 TEST CASES

**对应 PRD**：`docs/make-look-scanned-porting-PRD.md`  
**对应 ARCH**：`docs/make-look-scanned-porting-ARCH.md`  
**测试策略**：金字塔测试驱动  
**文档版本**：0.1.0  
**日期**：2026-06-23

## 1. 测试目标

本测试文档用于驱动 `make-look-scanned` 懒猫微服移植实现。测试重点不是验证上游 Go 效果算法本身，而是验证本次移植新增和改造的能力：

- Web 汉化。
- 本地 PDF 生成流程。
- PDF.js 和 WASM 本地资源加载。
- PDF 文件应用关联。
- 懒猫网盘打开。
- 懒猫网盘保存。
- 扫码友好预设。
- LPK 构建、本地安装和提审前验收。

## 2. 测试金字塔

```text
                 手工验收 / 上架核对
              E2E：浏览器 + 懒猫环境
          集成测试：PDF.js/WASM/网盘 API
      单元/组件测试：路径、参数、状态、文案
```

执行原则：

- 程序改造阶段优先跑单元/组件和本地浏览器验证。
- 应用关联和网盘保存完成后跑懒猫环境集成测试。
- LPK 阶段必须用正式包 `lzc-cli lpk install` 验证，不以开发部署代替。

## 3. 测试数据

### 3.1 基础 PDF

用途：验证基本生成流程。

要求：

- 1 页。
- 普通中文文本。
- 文件较小。

预期：

- 能生成扫描版 PDF。
- 输出文件可打开。
- 页面无错误。

### 3.2 多页 PDF

用途：验证逐页处理和进度。

要求：

- 3 到 5 页。
- 包含文本和简单图片。

预期：

- 状态显示“正在渲染第 N / 总页数 页”。
- 输出页数与输入一致。

### 3.3 含二维码 PDF

用途：验证扫码友好。

要求：

- PDF 中包含一个二维码。
- 二维码尺寸不小于常见文档扫码区。
- 内容可用手机扫码验证。

预期：

- 推荐预设生成后二维码可扫码。
- 清晰预设生成后二维码可扫码。
- 重度扫描预设显示风险提示。

### 3.4 网盘 PDF

用途：验证 `file_handler` 和 `/_lzc/files/home` 读取。

要求：

- 存放在懒猫网盘。
- 文件名包含中文和空格的样例至少 1 个。

预期：

- 右键打开能进入本应用。
- 路径编码/解码正确。
- 应用显示文件名并可生成。

### 3.5 非 PDF 文件

用途：验证错误处理。

要求：

- `.txt` 或图片文件。

预期：

- 本机选择时提示不是 PDF。
- 网盘关联不应主动声明非 PDF 类型。

## 4. 单元/组件测试

这些测试优先针对前端纯函数和 UI 状态，不依赖真实懒猫环境。

### 4.1 路径归一化

1. 输入 `/Docs/a.pdf`，输出 `/Docs/a.pdf`。
2. 输入 `Docs/a.pdf`，输出 `/Docs/a.pdf`。
3. 输入 `/_lzc/files/home/Docs/a.pdf`，输出 `/Docs/a.pdf`。
4. 输入 `/Docs/a.pdf.`，输出 `/Docs/a.pdf`。
5. 输入空字符串，返回空路径并触发错误提示。
6. 输入带中文和空格路径，保留原始路径语义。

### 4.2 文件类型判断

1. `type` 为 `application/pdf` 时接受。
2. `type` 为空但文件名以 `.pdf` 结尾时接受。
3. 文件名为 `.PDF` 时接受。
4. 文件名不是 PDF 时拒绝。
5. 传入空文件时给出明确错误。

### 4.3 参数映射

1. 推荐预设映射出 `dpi`、`skew`、`grayscale`、`paperTone`、`noise`、`blur`、`edgeShadow`、`jpegQuality`。
2. 清晰预设的 `noise` 和 `blur` 小于推荐预设。
3. 清晰预设的 `jpegQuality` 不低于推荐预设。
4. 重度扫描预设的 `noise`、`blur`、`edgeShadow` 大于推荐预设。
5. 传给 `mls.reset()` 的字段名必须是上游 WASM 接口要求的 camelCase。

### 4.4 Seed 生成

1. 未填写 seed 时，对同一文件生成稳定的 FNV-1a 64-bit 字符串。
2. 用户填写 seed 时优先使用用户输入。
3. seed 为空白字符串时走默认内容 hash。

### 4.5 UI 状态

1. WASM 加载前生成按钮禁用。
2. WASM 加载成功后生成按钮可用。
3. 未选择文件点击生成时提示“请先选择 PDF”。
4. 新选择文件时旧预览被清空。
5. 生成过程中禁用生成按钮。
6. 生成失败后恢复生成按钮。
7. 生成成功后显示下载和保存按钮。

### 4.6 汉化文案

1. 主标题为中文。
2. 主按钮为中文。
3. 参数标签为中文。
4. 处理中状态为中文。
5. 错误提示为中文。
6. 重度扫描预设的扫码风险提示为中文。

## 5. 集成测试

### 5.1 PDF.js 本地资源加载

步骤：

1. 禁止外网或断开 CDN。
2. 启动本地静态服务。
3. 打开应用。

预期：

- `pdf.mjs` 从本地路径加载。
- `pdf.worker.mjs` 从本地路径加载。
- 页面不请求 `cdn.jsdelivr.net`。
- PDF 可正常解析。

### 5.2 WASM 加载

步骤：

1. 打开页面。
2. 等待状态变为就绪。
3. 检查全局 `mls`。

预期：

- `wasm_exec.js` 加载成功。
- `main.wasm` 加载成功。
- `mls.reset`、`mls.addPage`、`mls.build` 可调用。

### 5.3 本机 PDF 生成

步骤：

1. 选择基础 PDF。
2. 使用推荐预设。
3. 点击生成。

预期：

- 进度按页显示。
- `mls.reset()` 调用一次。
- `mls.addPage()` 调用次数等于 PDF 页数。
- `mls.build()` 成功返回。
- 输出 Blob 类型为 `application/pdf`。
- 预览 iframe 显示 PDF。

### 5.4 多页 PDF 生成

步骤：

1. 选择多页 PDF。
2. 点击生成。

预期：

- 状态显示每一页进度。
- 输出 PDF 页数与输入一致。
- 处理完成后不残留异常状态。

### 5.5 网盘文件读取

步骤：

1. 在懒猫环境中打开 `/?file=/Test/a.pdf`。
2. 应用请求 `/_lzc/files/home/Test/a.pdf`。
3. 返回 PDF Blob。

预期：

- 应用自动载入文件。
- 文件名显示正确。
- 用户可直接点击生成。

异常预期：

- `404` 时提示文件不存在或读取失败。
- `403` 时提示权限不足或读取失败。
- 返回非 PDF 时提示文件类型错误。

### 5.6 网盘保存

步骤：

1. 生成扫描版 PDF。
2. 点击保存到懒猫网盘。
3. 选择目标路径。
4. 执行保存。

预期：

- 写入 `/_lzc/files/home/<target>`。
- 保存后执行读取或大小校验。
- UI 显示保存成功和路径。
- 网盘中能看到文件。
- 文件可打开。

### 5.7 文件选择器 inject

步骤：

1. 在 LPK 安装环境中打开应用。
2. 点击文件选择。
3. 触发懒猫文件选择器。

预期：

- 可选择本机文件。
- 可选择懒猫网盘 PDF。
- 选择后进入同一 `loadFile` 流程。

通过标准：

## 9. 代码核对测试

实现完成后逐项核对代码：

### 9.1 Web 入口

- 页面启动时先加载 PDF.js 和 WASM。
- 生成按钮初始禁用。
- 就绪后按钮启用。
- `file` URL 参数在启动时被读取。
- 文件读取统一进入 `loadFile(file, source)`。
- 所有状态提示通过统一 `status()` 或等效函数输出。

### 9.2 WASM 调用

- `mls.reset()` 在每次生成前调用。
- `mls.addPage()` 在每页渲染后调用。
- `mls.build()` 只在至少一页成功添加后调用。
- `r.error` 均被检查。
- 失败时抛出或显示中文错误。

### 9.3 Blob 生命周期

- 新生成前释放旧 `previewUrl`。
- 新选择文件时清空旧预览。
- 下载链接使用当前 Blob URL。
- 保存到网盘使用当前 Blob，不重新生成。
- 不在用户下载/保存前过早 revoke 当前 Blob URL。

### 9.4 网盘打开

- `file` 参数经过 `decodeURIComponent` 或等效安全处理。
- 路径不会出现 `/_lzc/files/home/_lzc/files/home` 重复前缀。
- fetch 失败时保留 HTTP 状态。
- 成功读取后文件名合理。

### 9.5 网盘保存

- 保存路径经过归一化。
- 文件名默认以 `.scanned.pdf` 结尾。
- PUT 使用 `application/pdf`。
- 保存成功后有校验。
- 保存失败不丢失生成结果。

### 9.6 Manifest 和权限

- `file_handler.mime` 包含 PDF MIME 和扩展兜底。
- `actions.open` 正确带 `%u`。
- `application.injects` 覆盖目标页面。
- `package.yml` 包含文档读写权限。
- `lzc-build.yml` 包含 `contentdir`。

## 10. 建议执行顺序

1. 完成程序改造后，先跑单元/组件核对。
2. 启动本地静态服务，跑本机 PDF、汉化、扫码样例。
3. 加入懒猫路径逻辑后，用 URL 参数模拟网盘打开。
4. 完成 LPK 配置后，在懒猫环境跑网盘右键打开。
5. 完成网盘保存后，跑保存和读取校验。
6. 构建镜像并同步到懒猫可拉取仓库。
7. 构建正式 LPK。
8. 本地 `lpk install`。
9. 跑完整 E2E 和上架前手工核对。
10. 上传提审。
